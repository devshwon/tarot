## Apps in Toss 콘솔 입력 템플릿 (초기 설정용)

> 이 템플릿은 **앱을 처음 만들 때** 콘솔에 입력해야 하는 정보를 정리하기 위한 것이다.  
> 현재 앱 **타로(tarot)** 기준으로 placeholder를 채워 두었다. 필요 시만 수정해서 사용한다.

---

## 1. 앱 정보

- **한국어 앱 이름**: `타로`
- **영어 앱 이름**: `Tarot`

- **appName**: `tarot`
  - 이 값은 미니앱을 구분하는 **고유 ID**이다.
  - `granite.config.ts`의 `appName` 값과 **완전히 동일**해야 한다.
  - 앱 스킴 `intoss://tarot/...` 호출 시 이 값을 기준으로 Apps in Toss에서 미니앱을 찾는다.

---

## 2. 카테고리 (최대 2개)

> 미니앱 홈에서 노출될 카테고리를 선택한다. **최대 2개까지** 선택 가능.
> (예: 음식·음료, 교육, 건강, 교통, 공공·행정, 소셜, 편의, 쇼핑, AI, 비즈니스, 콘텐츠, 여행, 일상, 정보 등)

- **1차 카테고리**: `콘텐츠`
- **2차 카테고리 (선택)**: `일상`

설명:
- 사용자가 이 서비스를 어떤 상황에서 떠올릴지 기준으로 가장 자연스러운 카테고리를 선택한다.

---

## 3. 노출 정보

### 3-1. 부제 (0~20자)

- **부제**: `매일 한 장 타로로 오늘의 운세를 확인`
- 조건:
  - 서비스를 대표하는 **짧은 설명**
  - **비속어, 느낌표(!) 금지**
- (20자 초과 시 예시) `데일리 타로, 오늘의 한 장` (14자)

### 3-2. 상세 설명 (0~500자)

> 사용자가 서비스에서 **무엇을 보고**, **어떤 버튼을 누르고**, **무엇을 경험하는지**를 구체적으로 적는다.

작성 템플릿:

1. 사용자는 **홈에서 "오늘의 한 장" 카드를 탭해 뒤집고**, 짧은 해석을 본다.
2. **"광고 보고 상세 해석" 버튼**으로 상세 해석을 잠금 해제하거나, **"질문하기"**에서 질문을 입력해 카드를 뽑고, **"기록"**에서 최근 14일 카드를 확인한다.
3. 그 결과로 **오늘의 타로 해석(짧은/상세), 질문별 카드 해석, 과거 기록**을 제공한다.
4. **"설정"**에서 면책 조항 확인, 데이터 초기화를 할 수 있다.

- **상세 설명 초안**:
  - 사용자는 홈 화면에서 오늘의 한 장 타로 카드를 탭해 뒤집고, 짧은 해석을 확인합니다. "광고 보고 상세 해석" 버튼을 누르면 상세 해석을 볼 수 있고, "질문하기"에서 마음속 질문을 입력하면 카드를 뽑아 짧은 해석과 조언을 받을 수 있습니다. "기록"에서는 최근 14일간 뽑은 카드를 확인할 수 있습니다. "설정"에서는 면책 조항을 읽고, 데이터 초기화로 저장된 기록을 삭제할 수 있습니다. 본 서비스는 오락 목적으로만 제공됩니다.

### 3-3. 앱 검색 키워드

- **키워드 목록**: `타로, 운세, 데일리 카드, 카드 뽑기, 질문하기, 오락`

가이드:
- 사용자가 앱을 검색할 때 떠올릴 만한 단어/문구를 쉼표(,) 또는 줄바꿈으로 구분해 적는다.

## 4. 앱 로고 제작용 프롬프트

> Apps in Toss 콘솔에 등록할 **앱 아이콘(로고)** 제작 시 아래 규칙과 프롬프트를 사용한다.

### 4-1. 로고 규칙

| 항목 | 규칙 |
|------|------|
| **배경색** | **필수**. 브랜드 대표 컬러 권장. **흰색·미색(베이지) 사용 금지**. |
| **형태** | 작은 크기에서도 식별 가능하도록 **최대한 단순한 형태**. |
| **그리드** | **원형(circle) 안에만** 디자인. 원 밖으로 나가면 안 됨. |
| **사이즈** | **600×600px** 로 제작. |

### 4-2. 로고 제작용 프롬프트 (복사해서 사용)

아래 블록을 복사한 뒤 필요 시 문구만 수정해서 이미지 생성 도구에 넣는다.

```
앱 아이콘(앱 로고) 이미지를 만들어 주세요.

- 주제/컨셉: 타로. 매일 한 장의 타로 카드로 운세를 확인하는 데일리 콘텐츠 미니앱.
- 스타일: 단순한 일러스트/심볼. 작은 크기(예: 48px)에서도 잘 보이도록 선과 형태가 단순하고 명확하게. (예: 카드 한 장, 별, 달, 크리스탈볼 등 단일 심볼)
- 배경: 원형(circle) 배경 필수. 배경색은 골드/금색(#D4A853) (흰색·미색·베이지 금지).
- 제약: 디자인 전체가 원 안에 들어가야 함. 원형 그리드 내부에만 요소 배치. 600×600px 비율의 정사각형 캔버스.
- 금지: 글자/텍스트 포함 금지, 복잡한 디테일 금지, 흰색/미색 배경 금지.
```

### 4-3. 입력 예시

- **주제/컨셉**: `타로. 매일 한 장의 타로 카드로 운세를 확인하는 미니앱`
- **배경색**: `골드(#D4A853)` (granite.config.ts의 brand.primaryColor와 동일)

### 4-4. 제작 후 확인

- [ ] 배경색이 **흰색·미색이 아니다**.
- [ ] 로고가 **원형 안에만** 들어가 있다.
- [ ] **600×600px** 로 내보냈다.
- [ ] 작게 줄여 봐도 **형태가 구분된다** (단순함).

---

## 5. 초기 체크리스트

- [ ] `granite.config.ts`에 `appName = tarot` 으로 설정했다.
- [ ] 카테고리(최대 2개)가 콘솔에 정확히 선택됐다.
- [ ] 부제(0~20자)와 상세 설명(0~500자)이 가이드에 맞게 작성됐다.
- [ ] 검색 키워드가 최소 3개 이상 정의됐다.
- [ ] 앱 로고가 규칙(배경색·원형·600×600px·단순 형태)에 맞게 제작됐다.
