## 출시 노트 + 앱 내 기능 스킴 템플릿 (개발 완료 후)

> 이 템플릿은 **기능 개발과 QA가 어느 정도 끝난 뒤**,  
> Apps in Toss 콘솔의 출시 노트·앱 내 기능 스킴을 작성하기 위해 사용한다.  
> 타로(tarot) 앱 기준으로 채워 두었다. 필요 시만 수정해서 사용한다.

---

## 1. 서비스 한 줄 설명

- 매일 한 장의 타로 카드로 오늘의 운세를 확인하고, 질문을 입력하면 카드 해석과 AI 상세 해석을 받을 수 있는 오락용 미니앱입니다.

---

## 2. 서비스 개요

- **타깃 사용자**: 타로·운세에 관심 있는 사용자, 데일리 카드로 하루를 시작하고 싶은 직장인·일상인
- **해결하려는 문제**: 오늘 하루에 대한 짧은 메시지나 고민에 대한 답변이 필요할 때 가볍게 참고할 수 있는 콘텐츠가 없음
- **핵심 가치**: 매일 한 장의 타로 카드로 짧은 해석을 보고, 광고 시청 후 상세 해석·AI 추가 해석을 확인하며, 질문하기로 마음속 고민에 대한 카드 해석을 받을 수 있음. 기록으로 최근 14일 카드를 다시 볼 수 있음

---

## 3. 주요 화면 및 기능 요약

| 구분 | 화면/기능 이름 | 경로(라우트) | 설명 |
|------|----------------|--------------|------|
| 홈 | 오늘의 카드 | `/` | 오늘의 한 장 타로 카드를 탭해 뒤집고, 짧은 해석을 확인. 광고 시청 후 상세 해석·추가 AI 해석 가능 |
| 기능 1 | 상세 해석 | `/detail` | 오늘의 카드 상세 해석, 조언, 행운 정보, 광고 시청 후 AI 추가 해석 |
| 기능 2 | 질문하기 | `/ask` | 질문 입력 후 카드 뽑기 → 짧은 해석·광고 시청 후 AI 상세 설명 |
| 기능 3 | 기록 | `/history` | 최근 14일간 뽑은 카드 목록. 항목 탭 시 해당 날짜/질문의 상세·AI 해석 확인 |
| 설정 | 설정 | `/settings` | 면책 조항 확인, 앱 데이터 초기화 |

---

## 4. 사용 시나리오 예시

1. 사용자는 **타로** 미니앱을 실행한다.
2. 홈 화면에서 **카드를 탭**해 뒤집고, **광고 보고 상세 해석 보기** 버튼을 눌러 상세 해석을 본다.
3. 하단 **질문하기** 탭에서 마음속 질문을 입력하고 **카드 뽑기**를 누르면, 카드와 짧은 해석이 나온다. **광고 보고 상세 설명 보기**로 AI 상세 해석을 받을 수 있다.
4. **기록** 탭에서 최근 14일 카드 목록을 보고, 항목을 탭하면 해당 카드의 상세·AI 해석을 다시 볼 수 있다.
5. **설정**에서 면책 조항을 읽고, 데이터 초기화로 저장된 기록을 삭제할 수 있다.

## 5. 앱 내 기능 스킴 목록 (여러 개 가능)

> Apps in Toss 콘솔의 **앱 내 기능** 입력을 위한 템플릿이다.  
> 스킴은 `intoss://{{appName}}/{{screenName}}` 형태로 통일한다.

### 5-0. 공통 정보

- **appName**: `misterytarot`
  - `granite.config.ts`의 `appName` 값과 동일해야 한다.

---

### 5-1. 스킴 1: 오늘의 타로(홈)

- **한국어 기능 이름**: 오늘의 타로
- **영어 기능 이름**: Today's Tarot
- **이동 URL (스킴)**:  
  `intoss://misterytarot/`

- **기능 설명 (콘솔 메모용, 선택)**:
  - 오늘의 한 장 타로 카드가 보이는 홈 화면으로 진입. 카드 탭 시 뒤집어 짧은 해석 확인 가능.

---

### 5-2. 스킴 2: 타로 해석(상세)

- **한국어 기능 이름**: 타로 해석
- **영어 기능 이름**: Tarot Reading
- **이동 URL (스킴)**:  
  `intoss://misterytarot/detail`

- **기능 설명**:
  - 오늘의 카드에 대한 상세 해석, 조언, 행운 정보 화면. 광고 시청 후 AI 추가 해석 요청 가능.

---

### 5-3. 스킴 3: 타로 질문하기

- **한국어 기능 이름**: 타로 질문하기
- **영어 기능 이름**: Ask Tarot
- **이동 URL (스킴)**:  
  `intoss://misterytarot/ask`

- **기능 설명**:
  - 질문 입력 후 카드 뽑기 화면. 카드·짧은 해석 표시 후, 광고 시청 시 AI 상세 설명 제공.

---

### 5-4. 스킴 4: 내 타로 기록

- **한국어 기능 이름**: 내 타로 기록
- **영어 기능 이름**: My Tarot History
- **이동 URL (스킴)**:  
  `intoss://misterytarot/history`

- **기능 설명**:
  - 최근 14일간 뽑은 카드 목록. 항목 탭 시 해당 날짜/질문의 상세·AI 해석 화면으로 이동.

---

### 5-5. 스킴 5: 타로 설정

- **한국어 기능 이름**: 타로 설정
- **영어 기능 이름**: Tarot Settings
- **이동 URL (스킴)**:  
  `intoss://misterytarot/settings`

- **기능 설명**:
  - 면책 조항 확인, 앱 데이터(기록 등) 초기화.

---

## 6. 정책/유의 사항 요약 (선택)

- **데이터/권한 사용**:
  - 카메라·위치·앨범 등 별도 권한 없음. 로컬 저장(localStorage)으로 최근 14일 카드 기록·해금 상태만 저장.
- **인앱 광고/결제**:
  - 리워드 광고 사용. "광고 보고 상세 해석 보기" / "광고 보고 추가 해석 보기" 등 노출 시, 시청 완료 시 상세·AI 해석 잠금 해제. 앱인토스 미지원 환경에서는 광고 없이 해석 제공(폴백).
- **기타**:
  - 모든 타로 해석은 오락 목적이며, 심리·영적·의학·재정 조언이 아니며 결정 근거로 사용할 수 없음. 설정 화면 면책 조항에 명시.

## 7. 최종 체크리스트

- [ ] 출시 노트에 **서비스 한 줄 설명**과 **주요 기능 요약**이 포함됐다.
- [ ] 최소 1개 이상의 앱 내 기능 스킴(`intoss://misterytarot/...`)이 정의됐다.
- [ ] 스킴으로 진입했을 때 화면/기능이 정상 동작하는지 실기기로 확인했다.
- [ ] 권한/인앱 광고/결제 등 정책 관련 설명이 누락되지 않았다.
