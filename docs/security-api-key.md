# API 키 노출 이슈 (보류)

## 요약

- OpenAI API 키(`VITE_OPENAI_API_KEY`)가 두 경로로 외부 추출 가능한 상태에 있음.
- 사용자 결정으로 **"문제 생기기 전까지 보류"**. 본 문서는 그 결정과 위험을 기록하기 위함.
- 결정일: 2026-04-30

---

## 노출 경로

### 1. git 히스토리

- `.env` 파일이 commit `d291e60` ("[1.0.3]광고 및 chatgpt api key 변경", 2026-04-04) 시점에 추가됨.
- 이후 `.gitignore`에 추가하고 `git rm --cached .env` 했으나, **과거 커밋에는 키가 그대로 남아 있음**.
- 리포지토리에 접근 권한이 있는 사람(또는 향후 공개로 전환 시 누구나)은 `git log -p -- .env` 로 키 추출 가능.

### 2. 클라이언트 번들

- `rsbuild.config.ts`에서 `loadEnv({ prefixes: ['VITE_'] })`로 환경변수를 클라이언트 번들에 주입.
- `src/services/gptTarot.ts`가 `import.meta.env.VITE_OPENAI_API_KEY`를 읽어 fetch 헤더에 그대로 사용.
- 빌드 산출물 `dist/web/static/js/index.*.js`에 키 문자열이 평문으로 박힘.
- 토스 미니앱 WebView로 이 JS가 전달되므로, 단말에서 디버깅 도구나 네트워크 캡처로 키 추출 가능.

---

## 왜 위험한가

- 추출된 키로 제3자가 OpenAI API를 호출 → **요금이 우리 계정에 청구됨**.
- 악용 패턴 예: 자동화된 호출로 일일 한도 소진, 다른 모델로 대량 호출, 외부 서비스에 키 재판매.
- 발견 시점에 따라 수십~수백만원 단위 청구 가능. OpenAI는 환불 보장하지 않음.

---

## 정기 교체가 필요한가

- **평상시:** 키가 한 번도 노출 경로(git/번들/로그/스크린샷 등)에 들어간 적 없다면 정기 교체는 필수 아님.
  OpenAI 자체 만료 정책과 사용량 알림으로 충분.
- **이번 케이스:** 위 두 경로 모두 노출에 해당하므로 "노출 사고" 처리가 표준.
  - 즉시 키 회전 → 새 키는 git/번들에 절대 들어가지 않는 구조로 변경.
  - 이번에는 사용자 판단으로 보류.

---

## 보류 중 모니터링 (최소 안전망)

키 회전을 미루는 동안 최소 다음은 유지/설정할 것.

- [ ] OpenAI 대시보드에서 **사용량 한도(soft/hard limit)** 를 평소 사용량의 1.5~2배로 설정.
- [ ] **사용량 알림 이메일** 활성화 (50% / 75% / 90%).
- [ ] 월 1회 OpenAI Usage 페이지에서 비정상 호출 패턴(시간대 분포, 모델 분포) 점검.
- [ ] 의심스러운 청구가 한 번이라도 보이면 **즉시** 아래 "사고 발생 시 절차" 진행.

---

## 사고 발생 시 절차 (런북)

1. **OpenAI 대시보드에서 해당 키 즉시 폐기 (revoke).**
2. 새 키 발급. **`.env` 외부에는 절대 두지 말 것** (Slack/메모/git 등 금지).
3. 클라이언트 직접 호출 구조를 폐기하고 프록시 경유로 전환:
   - `src/services/gptTarot.ts` 코드는 이미 `VITE_GPT_PROXY` 경로를 지원. (`gptTarot.ts:15-21`)
   - 프록시 서버에서만 키 보유. 클라이언트는 `VITE_GPT_PROXY` URL만 알면 됨.
   - `VITE_OPENAI_API_KEY` 코드 경로(`gptTarot.ts:8-13, 80-83, 115`) 제거.
4. git 히스토리 정리 (선택, 리포 공개/이관 예정 시 필수):
   - `git filter-repo --path .env --invert-paths` 로 모든 커밋에서 `.env` 제거.
   - force push 필요 → 협업자 모두에게 사전 공지.
   - 단순 폐기된 키라도 히스토리에 남아 있으면 자동 스캐너가 false alarm을 계속 트리거.

## 현재 상태 (2026-04-30 기준)

- `.gitignore`에 `.env`/`.env.*` 추가 (단, `.env.example`은 유지).
- `git rm --cached .env` 로 추적에서 제외. 로컬 파일은 유지.
- 이후 커밋에서는 더 이상 `.env`가 변경 추적 대상이 아님.
- **단, 과거 커밋에는 키가 그대로 남아 있음 (의도된 보류).**

## 참고: 클라이언트 번들에 키를 두는 구조 자체가 안티패턴

본 앱처럼 사용자 디바이스에서 실행되는 코드(웹/WebView/모바일 앱)는 어떤 난독화·압축을 거쳐도 결국 디바이스에서 평문으로 디코딩됨. 따라서:

- API 키, 시크릿, DB 자격증명 등은 **백엔드에만** 둔다.
- 클라이언트는 백엔드 프록시를 호출하고, 인증은 사용자 토큰(Toss 인증 등)으로 한다.
- 본 프로젝트에서는 `VITE_GPT_PROXY` 경로가 그 역할을 의도한 설계.
  키 회전 시 프록시 전환을 동시에 진행하는 것이 정석.
