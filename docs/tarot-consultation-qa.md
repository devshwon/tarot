# 타로 상담 — 실기기 QA 체크리스트

토스앱(WebView)에서 실제로 한 사이클을 끝까지 돌려 확인한다.
릴리스 직전 / 큰 변경 후 매번 진행.

## 사전 체크
- [ ] CF AI Gateway가 활성 상태 (대시보드 → AI Gateway → toss-tarot)
- [ ] `wrangler tail` 켜둔 채 검증 → upstreamUrl이 `gateway.ai.cloudflare.com`로 시작하는지 확인
- [ ] OpenAI Activity 또는 AI Gateway Analytics 탭에 호출 잡히는지 확인

---

## 1. Home에서 진입

- [ ] 첫 화면 **최상단에 "타로 상담"** 카드가 노출됨 (오늘의 카드 헤더보다 위)
- [ ] 오늘의 카드를 펼치지 않은 상태에서도 "상담 시작하기" 버튼 동작
- [ ] 오늘의 카드를 뽑은 뒤에도 동일하게 진입 카드 노출됨
- [ ] 카피에 "오늘", "더 깊게" 같은 일일 카드 연장선 표현 없음

## 2. 카테고리 선택 (`/consult`)

- [ ] 5개 카테고리 모두 표시 (연애·관계 / 금전·재물 / 진로·시험 / 인간관계 / 선택·미래)
- [ ] 각 카테고리 이모지 + 라벨 + 보조 힌트 한 줄 노출
- [ ] 탭하면 input 스테이지로 부드럽게 전환

## 3. 질문 입력

- [ ] 카테고리 카드(선택한 카테고리 정보) 보임
- [ ] 질문 입력 카드(테두리 있는 박스) 안에 TextArea 노출
- [ ] placeholder가 카테고리에 맞는 예시로 노출 (회전 — 다시 들어가면 다른 placeholder 가능)
- [ ] 글자수 카운터 N/100자 동작
- [ ] 2자 미만일 때 "카드 뽑기" 버튼 비활성
- [ ] 질문 2자 이상 입력 시 **스프레드 미리보기 배지** 노출 ("✨ 이 질문은 카드 N장 ...")
  - "오늘 ~해도 될까?" 입력 시 → "1장" 표시
  - "A 아니면 B?" 입력 시 → "2장" 표시
  - "그 사람은 어떨까?" 입력 시 → "3장" 표시
- [ ] **"💡 카드 장수는 어떻게 정해질까요?"** 카드가 펼쳐졌다 접혔다 동작
  - 펼치면 1장 / 2장 / 3장 각각 예시 2개씩 노출
- [ ] **자해 표현** 입력 시 ("자살하고 싶어요" 등) → 입력창 사라지고 안내 카드 + 핫라인(1393, 1577-0199) 노출
  - "다른 질문으로 바꾸기" 누르면 입력창 복귀, 질문 비워짐

## 4. 카드 뽑기

- [ ] 1장 질문 → 카드 1장 픽 화면
- [ ] 2장 질문 → "1/2 지금의 나" → "2/2 다른 길" 시리얼 픽
- [ ] 3장 질문 → "1/3 상황" → "2/3 조언" → "3/3 결과"
- [ ] 이미 뽑힌 카드는 흐릿하게 + 클릭 무시
- [ ] 같은 카드 두 번 탭으로 확정 동작
- [ ] "질문 다시 쓰기" 누르면 input 스테이지로 복귀

## 5. 결과 (광고 게이트)

- [ ] 카드 픽 직후: 카드 row(의미 없음) + **"광고 보고 결과 보기" / 해석권 보유 시 "해석권으로 결과 보기"** 노출
- [ ] "광고 보고 결과 보기" 누르면 광고 재생
  - 광고 시청 완료 → 카드별 의미 + 종합 해석 + follow-up 추천 2~3개 노출
  - 광고 닫으면 결과 잠금 유지 (다시 버튼 누를 수 있음)
  - 광고 실패 (3번) 시 → 광고 없이 결과 노출 (페널티 없음)
- [ ] 응답 지연 체감: 광고 보는 동안 fetch 시작되어 광고 끝났을 때 즉시 결과 또는 짧은 추가 로딩만
- [ ] 결과 화면에 "면책 조항은 설정에서 확인할 수 있습니다." 노출
- [ ] 5턴 도달 시 "마무리해 보세요" 추천 배너 노출

## 6. Follow-up / 직접 입력

- [ ] 추천 follow-up 버튼 누르면 **곧장 카드 픽 화면**으로 이동 (input 안 거침)
  - 직전 흐름이 LLM 시스템 메시지에 carry — 답변에 자연스럽게 연결되는지 확인 (예: "앞서 ○○ 카드를 고려할 때")
- [ ] "직접 입력해서 묻기" 누르면 **input 스테이지** 재진입
  - 상단에 **"직전 흐름"** 카드 노출 (이전 질문 + 카드명)

## 7. 마무리 (`closing` 스테이지)

- [ ] "오늘은 이만 듣기" 누르면 마무리 화면으로 전환
  - 카테고리 + 턴 수 + 카드 합계 + closingMessage 노출
- [ ] "다른 카테고리 보기" → 카테고리 선택 화면 복귀, 세션 클리어
- [ ] "메인으로" → Home 복귀, 세션 클리어

## 8. 기록 (`/history`)

- [ ] 카드 단발 기록 + **세션 항목**이 시간순 통합 정렬됨
- [ ] 세션 항목: 이모지(카테고리) + "○○ 상담" + 날짜 + N턴 + 첫 질문 16자
- [ ] 세션 항목 탭 → `/consult-detail` 진입
- [ ] 카드 단발 항목 탭 → `/detail` 진입 (기존 동작 유지)

## 9. 세션 상세 (`/consult-detail`)

- [ ] 헤더: 날짜 + "○○ 상담" + N번의 질문
- [ ] 턴별 (질문 → 카드 row + 의미 → 종합 해석) 시퀀스 표시
- [ ] closingMessage 있으면 마무리 카드 노출
- [ ] "기록으로 돌아가기" 동작
- [ ] 면책 한 줄 노출

## 10. 에러 / 엣지 케이스

- [ ] 네트워크 끊긴 상태에서 카드 픽 후 unlock → 친절한 에러 + "다시 시도" 버튼
- [ ] OpenAI 일시 503 → "잠시 후 다시 시도해 주세요" 메시지
- [ ] 광고 SDK 미지원 환경 (iOS Safari 외부 등) → 광고 자동 폴백 후 결과 노출
- [ ] 토스앱 백그라운드/포그라운드 전환 후 세션 상태 유지되는지

## 11. 데이터 초기화

- [ ] 설정 → 데이터 초기화 → 세션도 함께 삭제됨 (`tarot_consult_sessions` 키 prefix 일치 확인)
