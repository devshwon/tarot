# 타로 상담 (Tarot Consultation) — 기획 정리

> 신규 탭. 카테고리 기반 **세션형** 타로 흐름.
> 단발 1회 질문(`/ask`)과 달리, 한 카테고리 안에서 카드 뽑기 → 해석 → 파생 질문이 이어진다.
> 작성일: 2026-05-03

---

## 1. 컨셉 한 줄
**"카테고리 기반 세션형 타로 상담"** — 첫 질문은 사용자가 직접 입력, 두 번째 턴부터는 추천 follow-up 2~3개 + 직접 입력으로 흐름을 이어간다.

## 2. 네이밍 / 라우팅
- 탭/페이지명: **타로 상담**
- 라우트: `/consult`
- 진입: Home(`/`)에 **별개 섹션 카드**로 추가. 하단 탭 4개는 유지.
- **주의**: 오늘의 카드와 시각·카피 모두 분리 (연장선 느낌 금지)
  - 오늘의 카드를 **펼치지 않아도** 진입 가능해야 함 → 카드 뽑기 전(picker) 화면과 결과 화면 양쪽에 동등 노출
  - 카피에 "오늘", "더 깊게" 등 일일 카드와의 연결 표현 사용 금지

## 3. 카테고리 (5개로 확정)
| ID | 라벨 | 비고 |
|---|---|---|
| `love` | 연애·관계 | 속마음·궁합·관계 모두 흡수 |
| `money` | 금전·재물 | 투자 단정 금지 (가드레일) |
| `career` | 진로·시험 | 합격 단정 금지 |
| `social` | 인간관계 | 가족·친구·직장 |
| `choice` | 선택·미래 | A vs B, 다가올 흐름 |

## 4. 첫 질문 UX — 직접 입력 only

추천 질문 패널은 두지 않는다. 입력창의 **placeholder + 보조 힌트 한 줄**만으로 가이드.

### Placeholder 풀 (카테고리별 4~5개, 랜덤 노출)

**연애·관계 (5)**
1. 그 사람은 지금 나를 어떻게 생각할까?
2. 우리 관계는 앞으로 어떻게 흘러갈까?
3. 헤어진 그 사람과 다시 이어질 수 있을까?
4. 짝사랑하는 사람에게 마음을 표현해도 될까?
5. 우리 둘은 잘 맞는 사이일까?

**금전·재물 (4)**
1. 이번 달 재정 흐름은 어떨까?
2. 새로운 수입처가 생길 가능성이 있을까?
3. 지금 큰 지출을 해도 괜찮을까?
4. 올해 안에 재정 상황이 나아질까?

**진로·시험 (5)**
1. 이번 시험·면접의 흐름은 어떨까?
2. 지금 직장을 옮기는 게 좋을까?
3. 새 일을 시작하기에 적절한 시기일까?
4. 준비하는 자격증·입시는 어떤 결과로 흐를까?
5. 지금 커리어 방향이 나에게 맞을까?

**인간관계 (4)**
1. 요즘 멀어진 그 사람과의 관계는 어떻게 풀어가야 할까?
2. 지금 가족과의 갈등은 어떤 흐름일까?
3. 새로 만나는 사람들과 잘 어울릴 수 있을까?
4. 직장에서의 인간관계는 어떻게 풀릴까?

**선택·미래 (5)**
1. 지금 고민 중인 두 가지 선택, 어느 쪽이 나을까?
2. 다가오는 변화는 나에게 어떤 의미일까?
3. 지금 쉬어가는 게 맞을까, 밀고 나가야 할까?
4. 곧 찾아올 흐름은 어떤 모습일까?
5. 이 결정을 미루는 게 나을까?

### 카테고리별 보조 힌트 한 줄
- 연애·관계: "마음·관계·궁합 무엇이든 편하게 물어보세요"
- 금전·재물: "단정적인 투자 판단은 피해주세요"
- 진로·시험: "결과 단정보다 흐름을 보는 톤으로 답해드려요"
- 인간관계: "관계의 흐름과 조언 위주로 봐드려요"
- 선택·미래: "선택지나 다가올 흐름을 함께 살펴봐요"

## 5. 스프레드 룰 (질문 분류 기반 자동 결정)

LLM에 맡기지 않고 **클라이언트 룰**로 결정. 일관성·예측성·비용 모두 유리.

### 룰 우선순위
1. **2장 (비교)** — 패턴 매칭:
   - "A or B" / "A 와 B 중" / "둘 중" / "vs" / "어느 쪽"
   - 위치 의미: `현재(나)` / `대안(상대 또는 선택지 B)`
2. **1장 (단답)** — 패턴 매칭:
   - 어미: "~해도 될까", "~할 수 있을까", "가능할까", "괜찮을까", "맞을까"
   - 시간 한정어 동반: "오늘", "이번 주", "지금"
   - 위치 의미: `현재 흐름`
3. **3장 (흐름·조언, 기본값)** — 위 둘에 해당 안 되면 모두 3장
   - 위치 의미: `상황` / `조언` / `결과`

### 룰 함수 시그니처 (예정)
```ts
type Spread = { count: 1 | 2 | 3; positions: string[] };
function decideSpread(question: string): Spread
```

## 6. 세션 흐름

```
[카테고리 선택]
   ↓
[첫 질문 직접 입력] ← placeholder + 힌트만
   ↓
[스프레드 룰로 카드 N장 결정]
   ↓
[사용자가 카드 직접 뽑기]  ← 몰입 핵심
   ↓
[해석 + 다음 추천 질문 2~3개 + 직접 입력 + "이만 듣기"]
   ↓ (다음 질문)
[다시 카드 뽑기 → 해석] (직전 1~2턴 컨텍스트만 유지)
   ↓ (이만 듣기 또는 N=5턴 즈음 추천 트리거)
[마무리 한 장 + 세션 요약 1~2줄 → 기록 자동 저장]
```

## 7. LLM 호출 설계
**한 턴당 1회 호출**로 끝낸다. 응답에 다음 4개를 한 번에 받음:
- 카드별 의미 (간단)
- 종합 해석 (질문에 대한 답)
- 다음 추천 질문 2~3개
- (마무리 시) 세션 요약 1~2줄

컨텍스트는 **직전 1~2턴만** 유지 → 토큰/지연 최소화.

응답에 "앞서 뽑은 ○○ 카드를 고려할 때…" 같은 **연결 문구 1줄**을 자연스럽게 포함하도록 프롬프트.

## 8. 카드 중복 처리
한 세션 내 **이미 뽑힌 카드는 다음 턴에서 제외** (UX 일관성).
세션 종료 시 풀 초기화.

## 9. 기록(History) 통합
- 기존 `/history`를 **세션 단위 그룹**으로 표시 (카테고리 뱃지 + 첫 질문 요약 + 턴 수)
- 세션 펼치면 턴별 (질문 → 카드 → 답변) 시퀀스
- 공유는 **개별 답변 단위** 유지 (세션 통째 공유는 민감정보 노출 위험)
- localStorage 키: `tarot_consult_sessions`

## 10. 종료 UX
- 매 턴 하단에 **"오늘은 이만 듣기"** 항상 노출
- 5턴 도달 시 **"마무리 카드로 정리하기"** 부드럽게 추천 (강제 X)
- 마무리 화면: 세션 요약 + "다른 카테고리 보기" / "메인으로"

## 11. 가드레일 (헌장 준수)
- 카테고리별 단정 금지:
  - 금전: "수익이 납니다" ❌ → "지금 흐름은 ~한 신호" ✅
  - 진로: "합격합니다" ❌ → "준비한 흐름이 ~한 결로 보임" ✅
- 민감 키워드 감지 시 부드러운 안내:
  - 의료/자해/도박/극단 표현 → 면책 + 전문기관 안내 카피
- 모든 답변 끝에 면책 한 줄: "면책 조항은 설정에서 확인할 수 있습니다."

## 12. 기존 기능과의 차이 정리

| 항목 | 질문하기(`/ask`) | 타로 상담(`/consult`) |
|---|---|---|
| 형태 | 1회 단발 | 세션형 (멀티 턴) |
| 카테고리 | 없음 | 5개 중 선택 |
| 카드 수 | 고정 | 룰 기반 1/2/3장 |

| 컨텍스트 | 없음 | 직전 1~2턴 |
| 기록 | 단일 답변 | 세션 단위 그룹 |

## 14. P5 QA 자체 점검 요약 (2026-05-03)
